# Spec 00 — Overview

## Goal

Build a hybrid fraud detection platform that is explainable, reportable, and queryable, in an order
where every spec produces a working artifact the next spec consumes.

## Scope

Ten specs, implemented in numeric order. Each spec below states its **Goal**, **Scope**, **Contract**
(inputs/outputs other specs rely on), and **Acceptance criteria**.

| # | Spec | Produces |
|---|------|----------|
| 01 | Data pipeline | cleaned, scaled, split dataset |
| 02 | Hybrid model | `data/outputs/alerts.json` |
| 03 | Explainability | SHAP values + narrative per alert (added to alerts.json) |
| 04 | Reports | `data/outputs/reports/*.md` + `*.json` |
| 05 | RAG (ChromaDB) | queryable vector collection of alert narratives |
| 06 | MCP multi-agent | FastMCP server exposing grounded tools + router |
| 07 | Backend API | FastAPI REST surface over all of the above |
| 08 | Frontend | React dashboard consuming the backend |
| 09 | UI hardening | dark/violet theme, resilient chat, paginated alerts, signal breakdown |

## Non-goals

- No real-time streaming ingestion — this is a batch scoring demo.
- No user auth/multi-tenancy — single analyst persona for the hackathon demo.
- No production model registry — models are serialized with `joblib` to `intelligence/pipeline/artifacts/`.

## Cross-cutting rules

1. **English only** in code, comments, docs, commit content.
2. **No AI-tool attribution** anywhere (no "generated by", no assistant names) — this is the team's own code.
3. **Determinism for the demo**: every stage that could depend on an external LLM must have a
   deterministic fallback so the whole pipeline runs offline.
4. **One config source**: `intelligence/pipeline/config.py` holds every path, threshold, and hyperparameter.
   Nothing else hardcodes a path.
